HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Be careful while creating tags as it will autocreate end tags for you.
  • This is fun!

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A .css file is used to set style formatting consistency in the HTML file and you need to link them.

Git

  • Be sure to add "git" first in the command line.
  • git status: checks what branch we are currently on.
  • git checkout -b branch name (ex: feature/add-html): creates a new branch and switches to it.
  • git checkout main: switches back to the main branch
  • git add -A: Add changed files in working directory to staging area in Git. This commits all files in the working branch.
  • git commit: Commit to the repository.
  • -m "": Add this at the end of a commit to describe changes implemented.
  • git pull origin main: Pulls in the main branch before opening pull request.
  • git push origin feature/add-html: push up all of our changes to repository in GitHub
